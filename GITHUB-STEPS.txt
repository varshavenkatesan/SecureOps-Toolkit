GITHUB DEPLOYMENT - SIMPLE STEPS
=================================

Follow these steps EXACTLY to deploy your SecureOps Toolkit to GitHub:

STEP 1: CREATE GITHUB REPOSITORY
---------------------------------
1. Open your web browser
2. Go to: https://github.com/new
3. Fill in:
   - Repository name: SecureOps-Toolkit
   - Description: Educational Python-based security toolkit featuring File Integrity Monitoring, Web Vulnerability Scanner, and Penetration Testing tools
   - Visibility: Public (recommended) or Private
   - DO NOT check any boxes (no README, no .gitignore, no license)
4. Click "Create repository"
5. Leave the page open (you'll need the URL)


STEP 2: RUN POWERSHELL COMMANDS
---------------------------------
Open PowerShell and run these commands ONE BY ONE:

# Navigate to project directory
cd C:\SecureOps-Toolkit

# Initialize Git
git init

# Configure Git (replace with YOUR details)
git config user.name "Varsha Venkatesan"
git config user.email "your-email@example.com"

# Add all files
git add .

# Create initial commit
git commit -m "Initial commit: SecureOps Toolkit v1.0.0 by Varsha Venkatesan"

# Add GitHub remote (REPLACE YOUR_USERNAME with your actual GitHub username!)
git remote add origin https://github.com/YOUR_USERNAME/SecureOps-Toolkit.git

# Set main branch
git branch -M main

# Push to GitHub
git push -u origin main

TROUBLESHOOTING
---------------

Issue: "Git not recognized"
Solution: Install Git from https://git-scm.com/download/win

Issue: "Permission denied" or "Authentication failed"
Solution: 
- Make sure you're logged into GitHub in your browser
- Use GitHub Desktop instead: https://desktop.github.com
- Or set up SSH keys: https://docs.github.com/en/authentication

Issue: "Remote origin already exists"
Solution: git remote remove origin
Then add it again with the correct URL

Issue: "Repository not found"
Solution: Double-check the URL and your GitHub username
